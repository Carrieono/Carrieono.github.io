WOJ1003-Birth of Noah

Author: Carrie Date: 三月 26, 2020  0:49:53 Category: dailynote

WOJ1003-Birth of Noah
题目给了一个明确的家族数，很容易想到用C语言的结构体数组来表示，信息很明确，这道题卡在我的是，如何处理输入，在<stdio.h>中定义的EOF可用来判断输入结束，因为其值为-1，超过了0~127的ASCII码值，这样我写出了第一份代码：

#include<stdio.h>
struct peoples{
char name[12] ;
int age;
int gen;
};

int main() {

struct peoples family[13]= { 
    { "Adam",930,1 },
    {"Seth",912,2 },
    {"Enosh", 905, 3 },
    {"Kenan",910,4},
    {"Mahalabel",962,5},
    {"Jared",962,6},
    {"Enoch",365,7},
    {"Methuselah",969,8},
    {"Lamech",777,9},
    {"Noah",NULL,10},
    {"Shem",NULL,11},
    {"Ham",NULL,11},
    {"Japheth",NULL,11}
};

char name0[12];
char name1[12];
int j;
int p1=0, p2=0;
while(scanf("%s %s",&name0,&name1)!=EOF){

        for ( j = 0;j < 13; j++) {
            if (name0 == family[j].name)
            {
                p1 = j;
                break;
            }
        }
        for ( j = 0;j < 13; j++) {
            if (name1== family[j].name)
            {
                p2 = j;
                break;
            }
        }

        if(family[p1].gen==NULL||family[p2].gen==NULL)
            printf("No enough information\n");
        else if(family[p1].gen<family[p2].gen)
            printf("Yes\n");
        else
            printf("No\n");

        if(!family[p1].age || !family[p2].age)
        {
            printf("Not enough infomation\n");
            continue;
        }
        else if(family[p1].age > family[p2].age)
        {
                printf("Yes\n");
                continue;
        }
        else
                {
                    printf("No\n");
                    continue;
                }

}

return 0;

}

但是结果错误，问题在于debug的时候发现，比如说family[0]与第一次循环输入的”Adam”应该相同，但是我们忽略了接收”Adam”时从缓冲区中读取了12个字节的内容，也就是说name0的内容是“Adam\0？？？？？？？”，而family[0]应该是“Adam….\0”,这时候通过“==”来判定是不合适的，所以应该改用strcmp，因为strcmp在遇到“\0”就停止了（另一种停止的情况是在比较中第一次发现不相等就退出）。

当然还有一个遗留问题，我的代码结果其实是对的，但是测试还是没有通过，是因为一个很奇怪的输入问题，scanf并没有读取到把第二行送入的标识，所以第一行因为换行符得到执行，但第二行却得不到执行，需要额外输入一个回车！疑惑！！！！
